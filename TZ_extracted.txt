Техническое задание
Платформа для проведения онлайнвикторин и квизов
1. Цель проекта
Разработать вебприложение для создания и проведения онлайнвикторин в режиме реального времени с синхронизацией между участниками, таблицей лидеров и чатом.
2. Функциональные требования
2.1. Создание викторин
Создание викторины с указанием: названия, описания, правил, количества раундов.
Добавление вопросов с вариантами ответов (1–4 варианта, один правильный).
Настройка таймера для каждого вопроса (10–120 секунд).
Загрузка изображений или видео к вопросам.
Предварительный просмотр викторины перед запуском.
2.2. Режим игры в реальном времени
Запуск викторины в режиме «прямого эфира» (live).
Синхронизация начала раунда для всех игроков.
Отображение таймера обратного отсчёта для текущего вопроса.
Блокировка возможности ответа после истечения времени.
Автоматическое переключение на следующий вопрос после завершения таймера или ответов всех игроков.
2.3. Таблица лидеров
Динамическое обновление рейтинга в реальном времени.
Начисление очков:
за правильный ответ (балл = оставшееся время в секундах);
бонус за самый быстрый ответ;
штраф за неправильный ответ.
Отображение топ10 игроков в текущем раунде и за всю игру.
Визуализация прогресса игроков (полосы, прогрессбары).
2.4. Автоматическая проверка ответов
Мгновенная проверка ответа при выборе варианта.
Подсветка правильного и неправильного ответа.
Отображение статистики по вопросу после завершения раунда (сколько выбрали каждый вариант).
Фиксация времени ответа для расчёта баллов.
2.5. Чат во время игры
Текстовый чат для общения игроков в реальном времени.
Возможность отправки эмодзи и стикеров.
Модерация сообщений (фильтрация нецензурных слов).
Приватные сообщения между игроками.
2.6. Экспорт результатов
Выгрузка данных по итогам игры в форматах:
CSV (список игроков, баллы, ответы на вопросы);
Excel (с помощью Maatwebsite/Laravel-Excel);
PDF (сводный отчёт с диаграммами).
Экспорт статистики по вопросам (процент правильных ответов, среднее время ответа).
2.7. Администрирование
Управление викторинами (создание, редактирование, удаление).
Просмотр истории проведённых игр.
Назначение модераторов для чата.
Логирование действий пользователей.
3. Технические требования
3.1. Стек технологий
Backend: Laravel 10+
Realtime: Laravel Broadcasting + Pusher / Socket.io
Frontend: Livewire / Inertia.js + Alpine.js, Chart.js для графиков
База данных: MySQL 8+
Экспорт: Maatwebsite/Laravel-Excel
Кэширование: Redis
Очереди: Laravel Horizon (для фоновой обработки экспорта)
Медиа: Intervention Image для обработки изображений
3.2. API
RESTful API для:
управления викторинами и вопросами;
получения статистики и лидеров;
интеграции с внешними сервисами.
Аутентификация через Laravel Sanctum (токены).
3.3. Производительность
Кэширование данных викторины (Redis) для быстрого доступа.
Оптимизация запросов к БД (индексы, Eager Loading).
Обработка экспорта в фоновом режиме (очереди).
Поддержка до 1 000 одновременных игроков без задержек.
3.4. Безопасность
Защита от CSRF, XSS, SQLинъекций.
Валидация всех входных данных.
HTTPS для всех запросов.
Ограничение частоты запросов (rate limiting).
Защита от ботов (CAPTCHA при регистрации).
3.5. Доступность
Адаптивный дизайн (мобильные устройства, планшеты, ПК).
Соответствие WCAG 2.1 (контрастность, клавиатурная навигация).
